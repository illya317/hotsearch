# 抖音/小红书推荐算法调研

> 调研目标：研究抖音（Douyin/TikTok）和小红书（RED）的内容推荐算法核心机制，提取可借鉴思路用于优化 hotsearch 评分系统。
> 
> 信息来源：公开算法说明、平台运营资料、技术论文、第三方研究报告（截至 2025-2026 年）。

---

## 1. 评分机制

### 1.1 抖音：行为预测 × 价值权重

抖音的核心排序公式可简化为：

> **推荐优先级 = 预测用户行为概率 × 行为价值权重**

算法并不直接"理解"内容质量，而是建立**用户行为与内容特征之间的统计关联**，通过预估用户对候选视频产生各类行为（完播、点赞、评论、转发、关注等）的概率，乘以该行为的价值权重，得到最终优先级分数，分数高者获得更靠前的推荐位。

**关键指标权重排序**（官方及多方分析一致）：

| 指标 | 权重 | 说明 |
|------|------|------|
| **完播率 / 播放时长** | 最高 | 完播率是碾压性指标，40%-50%+ 为优秀，15%-20% 为大盘平均；前 3 秒留存率单独占约 40% 权重 |
| **转发/分享** | 高 | 跨平台分享是最强正向信号，表明内容值得传播 |
| **评论** | 中高 | 评论密度和速度反映内容话题性；深度互动（评论区的二次互动）权重更高 |
| **点赞** | 中 | 基础认可信号，但被动性强，权重低于播放和转发 |
| **复看率/重播** | 高 | 循环播放、反复观看表明强兴趣；带 Loop 设计的视频有优势 |
| **关注转化** | 高 | 从单条视频带来新关注，说明内容/创作者具有长期价值 |
| **负面信号** | 惩罚 | 快速划走、点击"不感兴趣"会直接降权；同一类内容连续 3 条后自动触发疲劳度降权 |

**2025 年升级要点**：
- 从"24 小时爆量逻辑"转向**7-15 天慢热型成长路径**，推流周期延长。
- 引入"动态评分 + 分级赛马"体系，创作者按历史表现分为 S/A/B/C 四级，S 级账号可获得双倍流量加持。
- 新增"互动质量评分"：评论点赞比、购物转化率等深度指标直接影响推荐力度。

### 1.2 小红书：CES 评分 + 多维度质量分

小红书以 **CES（Community Engagement Score，社区参与度评分）** 为核心，公式公开且稳定：

> **CES = 点赞数 × 1 + 收藏数 × 1 + 评论数 × 4 + 转发数 × 4 + 关注数 × 8**

CES 分数直接决定笔记能否从当前流量池晋级到下一级。评论和关注的权重显著高于点赞和收藏，体现了小红书对**社区互动深度**的重视。

**2026 年"星云 5.0"升级后的综合得分模型**：

> **推荐得分 = (基础质量分 × 0.3) + (用户互动分 × 0.4) + (商业价值分 × 0.2) + (社交关系分 × 0.1) − 违规扣分**

| 维度 | 占比 | 具体构成 |
|------|------|----------|
| 基础质量分 | 30% | 画质、文案通顺度、原创性、信息价值 |
| 用户互动分 | 40% | 深度阅读率、收藏转化、@好友数、CES 相关指标 |
| 商业价值分 | 20% | 广告主出价（仅限报备笔记）、购买转化 |
| 社交关系分 | 10% | 粉丝互动频次、群聊提及量 |

小红书的流量分配具有明显的**去中心化**特征：新账号的优质内容也有机会获得大量曝光，50% 以上的流量倾斜给千粉以下的素人账号。

### 1.3 对比总结

| 维度 | 抖音 | 小红书 |
|------|------|--------|
| 核心逻辑 | 行为预测模型，预估用户各类行为概率 | CES 社区参与度评分 |
| 最强信号 | 完播率/播放时长 | 评论、关注 |
| 内容偏好 | 娱乐性、情绪价值、即时爽感 | 实用性、信息密度、收藏价值 |
| 时间窗口 | 前 3 小时关键，但周期可延长至 7 天 | 前 1 小时关键，长尾推荐可达 180 天 |
| 去中心化程度 | 高（新创作者可一夜爆红） | 极高（素人账号获得过半流量） |

## 2. 冷启动

### 2.1 抖音：流量池赛马机制

抖音的冷启动采用**阶梯式流量池 + 实时赛马**：

1. **初始流量池**：新视频发布后，无论创作者粉丝量多少，系统都会分配 **100–500 次**基础曝光。
2. **实时筛选**：系统监测完播率、点赞率、评论率、转发率等核心指标。
3. **晋级条件**：
   - 前 3 秒留存率 ≥ 60%
   - 完播率 ≥ 35%–40%
   - 点赞率 ≥ 3%–5%
4. **流量池跃迁**：表现优异者进入下一级更大流量池（1 万 → 10 万 → 100 万+）。
5. **横向赛马**：每 10 分钟进行一次同类型内容的横向对比，数据更优者获得晋级资格。
6. **黑马机制**：前 30 分钟内 5 秒播放率超过 75% 的内容，会获得额外的曝光倾斜。

**冷启动关键洞察**：
- 抖音只考核系统**推荐的那一小批用户**的互动数据，外部刷量无效。
- 创作者分级体系影响初始流量池大小：S 级账号初始曝光更高，但内容本身的数据表现才是决定性因素。
- 推流周期延长至 7 天，普通账号第 7 天播放量可达第 1 天的 2.5 倍。

### 2.2 小红书：种子推荐 + 高质量评论优先

小红书的冷启动流程：

1. **智能审核**：AI 自动扫描内容合规性（响应时间 < 5 秒）。
2. **初始流量池**：
   - 新注册账号：50–100 次展示
   - 正常权重账号：300–500 次展示
   - 高权重账号：800–1200 次展示
3. **标签匹配**：系统通过 NLP 提取笔记标签，匹配对此感兴趣的用户群。
4. **CES 评分驱动**：根据初始用户的互动数据计算 CES，决定是否进入更大流量池。

**小红书冷启动的独特之处**：
- **高质量评论优先**：冷启动阶段系统更关注高质量评论的数量，而非单纯的点击率或停留时长。高质量评论反映了内容是否被准确分发到了目标人群中。
- **预测评分机制**：在真正曝光前，系统会从过往笔记数据库中找出类似笔记，根据历史互动数据**预测**该笔记的评分，提前决定初始排名。
- **搜索权重极高**：超过 65% 的流量来源于搜索，搜索页的排序取决于关键词匹配度和短时间内互动量。

### 2.3 对比总结

| 维度 | 抖音 | 小红书 |
|------|------|--------|
| 初始曝光 | 100–500 次（相对统一） | 50–1200 次（按账号权重分层） |
| 核心筛选指标 | 完播率、3 秒留存率 | 高质量评论数、CES 评分 |
| 竞争机制 | 横向赛马（每 10 分钟） | 阶梯式晋级（按 CES 阈值） |
| 新内容优势 | 黑马机制（高播放率倾斜） | 去中心化分发（素人获流量倾斜） |
| 时间敏感性 | 前 3 小时决定 80% 命运 | 前 1 小时关键，但长尾极长 |

## 3. 探索与利用

### 3.1 抖音：兴趣图谱 + 疲劳度调控

抖音作为典型的**兴趣图谱（Interest Graph）**平台（而非社交图谱），其探索与利用平衡策略：

- **主逻辑**： exploitation 占主导，通过深度学习模型不断收紧用户兴趣画像，推送"最可能观看"的内容。
- **疲劳度调控**：同一类型内容连续出现 3 条后，算法自动降权，强制引入多样性内容。
- **探索类指标**：2025 年算法增强了对"搜索行为"的权重，用户主动搜索过的内容曝光量提升 3 倍。平台通过鼓励搜索来挖掘用户的潜在兴趣。
- **跨品类探索**：算法会测试将内容推送给相邻兴趣圈层的用户，若表现良好则逐步扩大。
- **地域/场景探索**：利用地理位置、设备状态、时间等上下文信号进行局部探索。

### 3.2 小红书：SSD 多样性算法 + 长期粘性优先

小红书的探索策略更注重**社区生态健康度**：

- **SSD（Sliding Spectrum Decomposition，滑动频谱分解）**：小红书自研的多样性算法，专门解决长序列浏览中的同质化问题。它通过张量分解捕捉用户在整个浏览序列（而非仅当前窗口）中的多样性需求，已被部署在生产环境的 Explore Feed 中。
- **北极星指标导向**：当短期指标（如点击率）与长期指标（如用户使用时长、留存）冲突时，**以长期指标为准**。这意味着牺牲部分即时点击率来换取多样性探索是被鼓励的。
- **流量倾斜机制**：主动将 50%+ 的流量分配给千粉以下素人账号，防止头部创作者垄断流量，维持内容生态的活力。
- **"一键破茧"功能**：2025 年平台上线"茧房评估"和"一键破茧"功能，让用户主动突破信息茧房，算法响应用户的破茧请求。
- **长尾召回**：发布 2–3 个月的老笔记也会被算法重新召回推送，优质内容的生命周期极长。

### 3.3 对比总结

| 维度 | 抖音 | 小红书 |
|------|------|--------|
| 核心策略 | 兴趣图谱精准匹配 + 疲劳度调控 | 社区生态平衡 + 长期粘性优先 |
| 多样性算法 | 疲劳度阈值（3 条同类降权） | SSD 滑动频谱分解（序列级多样性） |
| 新内容探索 | 黑马机制 + 相邻兴趣圈层测试 | 素人流量倾斜 + 长尾召回 |
| 用户自主权 | 较低（算法主导） | 较高（"一键破茧"等主动功能） |
| 优化目标 | 单次会话时长最大化 | 用户长期留存 + 社区活跃度 |

## 4. 标签/分类使用

### 4.1 抖音：三级标签 + 多模态特征识别

抖音的标签体系在 2025 年升级为**三级标签机制**：

| 标签层级 | 权重 | 来源 |
|----------|------|------|
| **核心标签** | 50% | 视频内容的高频关键词、视觉特征（AI 图像识别）、音频特征（声纹分析） |
| **辅助标签** | 30% | 用户搜索行为、评论区提炼的高频词 |
| **潜力标签** | 20% | 基于相似账号数据预测的扩展标签 |

**多模态内容理解**：
- **文本特征**：NLP 解析标题、字幕、口播文案中的关键词。
- **视觉特征**：图像识别提取画面中的物体、场景、人物、品牌 Logo 等。
- **音频特征**：声纹分析捕捉语音讲解中的关键词频率、背景音乐风格。

抖音已**几乎不依赖传统的用户/内容打标签**方式，而是通过神经网络直接预测用户行为。但标签仍在冷启动和候选生成阶段发挥重要作用。

### 4.2 小红书：内容标签 + 用户标签双向匹配

小红书的推荐算法仍然以**"标签匹配"**为主：

- **内容标签**：笔记发布后，系统通过 NLP 技术提取标题、正文、标签（hashtag）中的关键词，为笔记打上多维度标签。
- **用户标签**：基于用户的历史浏览、搜索、互动、收藏行为，构建超 2000 维的兴趣标签画像。
- **双向匹配**：将内容标签与用户兴趣标签进行匹配，决定初始推荐人群。

**搜索 + 推荐互通**（小红书独有优势）：
- 小红书 65%+ 流量来自搜索，因此**关键词匹配**是核心能力。
- 用户搜索某个关键词后，推荐页（发现页）会优先展示相关内容。
- 搜索结果排序取决于：① 关键词匹配程度；② 短时间内互动量。
- 这意味着内容创作者需要在标题、正文、标签中自然融入用户会搜索的关键词。

**三层推荐系统架构**：
1. **内容理解层**：视觉 AI 分析（构图、色彩、产品检测）、文本挖掘、多标签分类、质量评分。
2. **用户偏好层**：行为信号、兴趣图谱（带时间衰减）、社交关系、商业意图。
3. **匹配排序层**：协同过滤、基于内容的匹配、多样性注入（SSD）、实时 CTR 预测。

### 4.3 对比总结

| 维度 | 抖音 | 小红书 |
|------|------|--------|
| 标签核心作用 | 候选生成 + 冷启动匹配 | 贯穿推荐全流程的核心匹配逻辑 |
| 标签层级 | 三级标签（核心/辅助/潜力） | 内容标签 + 用户标签双向匹配 |
| 内容理解 | 多模态 AI（文本/视觉/音频） | 多模态 AI + 搜索语义理解 |
| 搜索权重 | 中（搜索占比逐渐增加） | 极高（65%+ 流量来自搜索） |
| 关键词策略 | 话题挑战 + 热门音乐/特效 | 标题/正文/标签自然融入搜索词 |

## 5. 反馈循环

### 5.1 抖音：100+ 行为信号的分钟级闭环

抖音的反馈循环以**密度高、实时性强**著称：

**显式信号**（用户主动行为）：
- 点赞、评论、转发、分享、关注、收藏
- 点击评论区、进入创作者主页
- 搜索相关内容、复访收藏的视频
- 负面反馈：点击"不感兴趣"、举报、快速划走

**隐式信号**（用户被动行为，权重往往更高）：
- **观看时长 / 完播率**：毫秒级精度的停留时间记录
- **重播 / 循环播放**：强兴趣信号
- **暂停、回放**：表明内容有值得反复观看的价值
- **划走速度**：快速划走 = 负向信号
- **部分点赞**（双击但未完成）：微交互信号
- **设备状态**：观看时的设备倾斜角度、是否充电、时间等

**反馈循环机制**：
- **分钟级实时更新**：抖音称已实现"分钟级"的推荐策略更新，用户的每一次滑动都会微调后续推荐。
- **跨平台数据融合**：抖音还会利用字节跳动其他产品（如今日头条）的跨平台行为数据。
- **A/B 测试常态化**：新策略通过小流量实验验证，数据达标后全量上线。

### 5.2 小红书：社区互动驱动的长周期反馈

小红书的反馈循环更强调**社区氛围和长期价值**：

**核心行为信号**：
- **点赞**：基础认可，权重最低（1 分）。
- **收藏**：实用性认可，权重中等（1 分）；收藏后的复访率是关键延伸指标。
- **评论**：社区互动核心，权重高（4 分）；**高质量评论**的权重尤其高。
- **转发**：传播意愿，权重高（4 分）。
- **关注**：长期价值认可，权重最高（8 分）。

**深度阅读信号**：
- 阅读完成率（滑动到底次数 / 点击次数，按笔记长度归一化）。
- 停留时长：图文 ≥ 25 秒，视频 ≥ 1 分 10 秒为达标。
- @好友数：社交传播信号。

**负面信号与治理**：
- 点击"不喜欢"、举报、快速划走。
- 2025 年上线"真诚分"体系，打击虚假和低质内容。
- 同质化搬运内容会被合并展示或直接限流。

**反馈循环机制**：
- **阶梯式放大**：CES 评分达标 → 进入更大流量池；不达标 → 逐渐减少推荐直至停止。
- **长尾召回**：优质笔记的推荐周期可达 180 天以上，算法会定期将老内容重新匹配给新用户。
- **搜索反馈互通**：用户在搜索页的行为会实时影响推荐页的内容分布。

### 5.3 对比总结

| 维度 | 抖音 | 小红书 |
|------|------|--------|
| 最强正向信号 | 完播率、播放时长、重播 | 高质量评论、关注、收藏 |
| 实时性 | 分钟级更新 | 小时级更新（前 1 小时关键） |
| 负向信号处理 | 快速划走、"不感兴趣"即时降权 | 同质化合并、真诚分降权 |
| 长周期反馈 | 7 天推流周期，老内容偶发 resurgence | 180 天+ 长尾推荐，主动召回老内容 |
| 社交属性 | 弱（兴趣图谱为主） | 强（社区互动为核心） |

## 6. 可借鉴的点（针对 hotsearch 评分系统）

结合抖音和小红书的算法实践，以下思路可直接或间接应用于 hotsearch 的热榜内容评分系统：

### 6.1 评分机制优化

**借鉴抖音的"行为预测 × 价值权重"模型**：
- 当前 hotsearch 系统已有基于标签的偏好权重分（`ScoringService`），可进一步**显式定义不同互动行为的价值权重**。
- 参考 CES 公式的加权思想，为不同平台设计差异化权重。例如：
  - 知乎：回答质量分 > 赞同数 > 评论数（知乎评论权重应高于微博）
  - 微博：转发数 > 评论数 > 点赞数（微博的传播属性）
  - GitHub：Star > Fork > Watch（开发者的实际使用行为）
- 引入**完读率/阅读完成率**作为隐性质量指标：对于需要跳转阅读的内容（如知乎回答、AI 新闻），可通过摘要的阅读完成度或原文点击率间接衡量。

**借鉴小红书的"基础质量分"概念**：
- 在现有标签打分基础上，增加**内容质量分**维度：
  - 标题信息密度（是否包含核心关键词）
  - 来源权威性（官方媒体 vs. 自媒体）
  - 时效性（发布时间与当前时间的衰减）
- 公式参考：`最终得分 = (基础质量分 × 0.3) + (互动加权分 × 0.4) + (标签偏好分 × 0.3)`

### 6.2 冷启动策略

**借鉴流量池思想解决新内容曝光问题**：
- hotsearch 采集的热榜内容本身就是平台筛选后的结果，但仍存在**平台内不同内容的热度差异**。
- 对于**新兴话题**或**跨平台首次出现的内容**，可设置"冷启动保护期"：
  - 首次出现的话题给予基础曝光权重（如小红书的初始流量池）。
  - 监测初期互动数据（点击率、阅读完成率），表现优异者提升评分。
- 引入**黑马机制**：对于在短时间内（如 1 小时内）互动增速异常高的内容，给予额外的评分加成，捕捉突发性热点。

**借鉴小红书的"预测评分"**：
- 在内容真正进入评分流程前，基于历史相似内容的表现**预估值**，避免全新类型内容被系统性低估。
- 例如：一条关于"新发布 AI 模型"的 GitHub Trending _repo_，可参考历史上同类_repo_的 Star 增速曲线进行预测打分。

### 6.3 探索与利用的平衡

**借鉴疲劳度调控机制**：
- hotsearch 目前按平台和标签聚合，容易出现**同一类内容连续推送**的问题（如连续多条 AI 新闻）。
- 在飞书推送和 API 返回结果中引入**多样性约束**：
  - 同一标签的内容连续出现不超过 N 条（参考抖音的"3 条降权"）。
  - 在高分内容中强制插入一定比例的低分但高多样性内容。

**借鉴小红书的 SSD 多样性思想**：
- 在 API 的 `/trends` 接口中，不仅按分数排序，还可在排序中注入**多样性项**：
  - 使用标签相似度计算内容间的"距离"。
  - 采用滑动窗口策略，确保返回列表中相邻内容的标签差异度不低于阈值。

**流量倾斜给"素人"内容**：
- 小红书将 50%+ 流量给千粉以下账号，避免头部垄断。
- hotsearch 可设置**来源多样性权重**：小众平台（如东方财富某细分板块）的优质内容，其评分可获得一定加成，避免知乎/微博等大平台垄断热榜。

### 6.4 标签/分类体系升级

**借鉴抖音的三级标签机制**：
- 当前 hotsearch 的 `TAG_RULES` 为静态关键词匹配，可升级为**动态权重标签体系**：
  - **核心标签**（50%）：通过 LLM + 关键词提取的标题主题标签。
  - **辅助标签**（30%）：从用户搜索历史、评论高频词中提取。
  - **潜力标签**（20%）：基于相似内容的共现标签预测。
- 标签权重随时间动态衰减和更新，而非固定不变。

**借鉴小红书的"搜索 + 推荐互通"**：
- hotsearch 用户通过飞书 Bot 发送的查询（如 "AI"、"股票"）可视为**搜索行为**。
- 将用户的飞书查询历史纳入评分权重：用户近期搜索过的标签，在后续推送中提高该标签内容的曝光概率。

**关键词自然融入**：
- 在 SummaryAgent 的 Jinja2 模板中，确保生成的摘要自然覆盖核心关键词，既利于人工阅读，也利于后续的标签匹配精度。

### 6.5 反馈循环增强

**建立更细粒度的反馈采集**：
- 当前系统主要依赖平台的公开数据（点赞、评论数等），但缺乏**用户对 hotsearch 推送内容的直接反馈**。
- 可借鉴抖音的显式/隐式信号分层，在飞书交互层增加：
  - **显式**：用户点击"展开详情"、收藏消息、回复关键词继续查询。
  - **隐式**：消息的阅读率（飞书已读状态）、用户在消息上的停留时间。
- 将这些反馈回传到 `preference.json`，形成真正的**闭环优化**。

**长周期内容召回**：
- 借鉴小红书的 180 天长尾推荐，hotsearch 可对历史高分内容进行**周期性召回**：
  - 每周/每月回顾过去的高分内容，若当前有类似事件触发（如某公司再次登上热榜），自动关联展示历史背景。
  - 在 API `/daily?period=24h` 基础上增加 `/weekly` 或 `/trending-comeback` 接口。

**分钟级热榜更新**：
- 抖音的分钟级更新提示我们，热榜的时效性至关重要。
- 当前 cron 任务为定时调度，可在高优先级来源（如微博、知乎）上增加**更短周期的采集频率**（如 15 分钟），或增加事件触发机制（当某内容互动增速超过阈值时即时采集）。

### 6.6 可直接落地的优化清单

| 优先级 | 优化项 | 借鉴来源 | 实施难度 |
|--------|--------|----------|----------|
| 高 | 显式定义各平台互动行为权重（转发>评论>点赞） | 抖音 + 小红书 CES | 低（调整配置） |
| 高 | 推送内容增加多样性约束（同标签连续出现 ≤2 条） | 抖音疲劳度调控 | 中（改 SummaryAgent） |
| 中 | 冷启动保护期 + 黑马增速检测 | 抖音流量池赛马 | 中（改 ScoringService） |
| 中 | 用户飞书查询历史影响后续推送权重 | 小红书搜索推荐互通 | 中（改 bot.py + preference） |
| 中 | 长周期内容召回机制（老热点关联） | 小红书长尾推荐 | 低（改 summary 模板） |
